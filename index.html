如何使用Github-Pages和Hexo搭建个人Blog

发表于 2022-03-29

第一篇blog写自己如何搭建blog :D 边搭边写，所以可能有点混乱

需求

  • Github账号（应该有吧）
  • Git
  • Node.js
  • Hexo

由于个人使用习惯，本文章适用于Ubuntu20.04(WSL2)

安装

首先是Git sudo apt install git 提示我已经安装了。。。

然后是Node.js 巨硬给了教程 在适用于 Linux 的 Windows 子系统 (WSL2) 上安装 Node.js

这里出了点问题，我发现wsl没法用Windows的代理以至于我无法从Github上clone东西,不是curl(7)就是curl(35)

但我发现，既然我的目的是搭blog，那我为什么要费力气装nvm啊

以下指令来自https://github.com/nodesource/distributions (这里不使用apt进行安装因为apt给的版本好像有点老)

$ curl -fsSL https://deb.nodesource.com/setup_17.x | sudo -E bash
$ sudo apt-get install -y nodejs

解决！

接下来是Hexo sudo npm install -g hexo-cli

我试了下不加sudo，提示我没权限。。。

好像也可以局部安装 npm install hexo

搭建

工作环境

在完成上述安装过程后，执行下述命令

$ hexo init <folder>
$ cd <folder>
$ npm install

新建的文件夹内有以下文件/文件夹

.
├── _config.yml //网站的配置信息
├── package.json //应用程序的信息
├── scaffolds //模版文件夹，后文的布局就保存在这个文件夹内
├── source //资源文件夹
|   ├── _drafts //我搭建的时候并没有这个文件夹
|   └── _posts //文章的生成位置
└── themes //主题文件夹，下载的主题要放在这里

各文件/文件夹详细用途建议查阅Hexo官网(虽然协议是CC BY 4.0，但大量复制粘贴感觉非常怪异)

接下来输入 hexo g ( hexo generate 的缩写) 和 hexo s ( hexo server 的缩写) ,然后就可以在 http://localhost:4000/ 上看见新搭出来的网站

现在网站是创建了，但全部都是默认配置，显然不能说是个人blog，所以我打算对其进行一些美化和调整

主题

主题可以在 https://hexo.io/themes/ 中找到(当然也能百度)

本文章使用Next示范(我的blog可能不会使用这个主题)

$ git clone https://github.com/theme-next/hexo-theme-next themes/next

啊，又是绕不过的网络问题，我去想想怎么办 好吧看上去需要用一点运气(反正能连上就是成功(╹ڡ╹)

接下来需要在 _config.yml 中修改一些配置来启用主题

themes: landscape
替换为
themes: next

language: en
替换为
language: zh-CN

新界面大概长这样

我不知道John Doe是谁 根据我百度到的信息，类似于张三或者李四，反正是个无名氏

配置

我不怎么想写配置，毕竟 https://hexo.io/zh-cn/docs/configuration 讲的比我详细多了，照着这个改基本上都没什么问题

themes/next/docs/zh-CN 这个目录下有Next的官方文档，如果想使用搜索功能、LeanCloud或者数学公式搜索引擎之类的可以自行查看

Next没有自带评论功能，所以如果想使用评论需要自己找，比如Gitalk或者Valine等等 先放着，过几天再写然后就忘了

写作

$ hexo new [layout] <title>

此命令会生成一篇布局为layout(如未指定则默认为post，当然可以在_config.yml里调)，标题为title的Markdown文件

有的时候文章里可能会需要添加图片或者其他东西，首先可以把图片丢进source/images，然后用 ![](/images/image.jpg) 进行引用

但这样会使images文件夹里面非常乱，这时可以在_config.yml中将 post_asset_folder: 的值修改为true，这样每次执行上面的指令之后都会生成一个同名文件夹，然后把文件丢进去就可以使用markdown语法进行调用

我在写的时候发现了一个问题,图片无法正常加载，我一开始查到的资料是使用 hexo-asset-image ，然后发现这个插件转换的图片路径不对，会多一个/.com/

然后我找到了一个插件 hexo-image-link ，工作原理是

![image file label](./markdown-file-name/local-image.png) -> {% asset_img label local-image.png %}

即在Hexo启用 post_asset_folder: true 选项后，将Markdown语法的图片路径转换为asset_img的方式，使图片能够在使用typora编辑和hexo预览发布时都能正常显示

$ npm install hexo-image-link --save

部署

终于写到这里了，在之前的需求中提到了Github账号和Git，现在是用到这些东西的时候了

Hexo配置文件中和部署相关的部分是deploy，一开始的情况是这样

# Deployment
## Docs: https://hexo.io/docs/one-command-deployment
deploy:
  type: ''

至少要有一个type参数，比如

deploy:
  type: git

当然type参数可以不止一个

deploy:
- type: git
  repo:
- type: heroku
  repo:

不过由于这次使用的是Github-Pages，所以只考虑Git

首先打开GitHub并创建一个仓库，仓库名必须是 <你的用户名>.github.io 比如

然后修改_config.yml

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
  type: git
  repo: 
    github: 你的仓库地址(后面要有.git)
  branch: master

接着安装hexo-deployer-git

$ npm install hexo-deployer-git --save

之后进行Git账户设置，设置过的可以跳过这部分

$ git config --global user.email "you@example.com"
$ git config --global user.name "Your Name"
$ ssh-keygen -t rsa -C "you@example.com"

密码设不设置看心情

在设置的目录里找到.ssh文件夹，里面有id_rsa和id_rsa.pub，复制id_rsa.pub里面的所有内容

回到GitHub，选择 “Settings” -> “SSH and GPG keys” -> “New SSH key”，取个标题，将你复制的所有内容丢进Key里面，然后单击 “Add SSH key”

最后部署到你的GitHub仓库

$ hexo cl && hexo g && hexo d

大功告成！现在可以在https://<你的用户名>.github.io访问新博客了
